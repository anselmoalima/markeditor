# ADR-001: pnpm + Turborepo monorepo with tsup dual ESM/CJS build

## Status

Accepted

## Date

2026-05-24

## Context

The `bob-editor` package must ship as a public NPM artifact that consumers can import from React 18 and React 19 applications running under Next.js (App Router and Pages), Remix, Vite, and legacy CRA setups. PRD §6.6 mandates a single published package (`packages/bob-editor`) plus a non-published test app (`apps/playground`), dual ESM/CJS output, complete `exports` map, peer-dependency declarations for React, source maps, type declarations, CSS bundling, `provenance` publication, and CI matrix coverage across Node 18/20/22 × React 18/19.

The build tool must produce ESM + CJS + `.d.ts` + bundled CSS in one pass, support subpath exports for `./styles` and `./plugins/*`, and run inside a CI pipeline that respects task dependency order across the workspace.

## Decision

Adopt a pnpm-workspaces monorepo orchestrated by Turborepo. The published library lives at `packages/bob-editor`. The Vite playground lives at `apps/playground` and consumes the library via `"bob-editor": "workspace:*"`. Builds run through `tsup` (Rollup-based) configured for dual ESM + CJS outputs, declaration generation via `rollup-plugin-dts`, CSS bundling to `dist/styles.css`, and source maps. Subpath entrypoints (`./plugins/*`) are declared as separate `tsup` entry points so each subpath is independently tree-shakable. Versioning is governed by Changesets; releases run from GitHub Actions with OIDC-backed `npm publish --provenance --access public`.

## Alternatives Considered

### Alternative 1: Single-package repo + Rollup

- **Description**: Skip workspaces; keep the library at the repo root and host the demo in a `demo/` folder built with the same Rollup config.
- **Pros**: Simpler tooling surface; fewer config files.
- **Cons**: No clear publication boundary; demo bundling pollutes the published artifact; harder to deploy the playground independently.
- **Why rejected**: PRD §7.2 explicitly mandates a monorepo with a separate, non-published playground, mirroring tiptap/lexical/mdxeditor conventions.

### Alternative 2: pnpm workspaces + unbuild

- **Description**: Use `unbuild` instead of `tsup` for the library build.
- **Pros**: Better default handling of mixed JS/CSS assets in some cases.
- **Cons**: Less mature CSS bundling story; rollup-plugin-dts integration requires hand-rolled steps.
- **Why rejected**: `tsup` is explicitly named in PRD §15 and has a battle-tested dual ESM/CJS + CSS path.

### Alternative 3: pnpm workspaces + Vite library mode for the library

- **Description**: Build the library with Vite's library mode plus `vite-plugin-dts`.
- **Pros**: Single tool across library and playground.
- **Cons**: Vite library mode is harder to configure for multiple entry points (`./plugins/*`) and produces noisier CJS output.
- **Why rejected**: Multi-entry subpath exports and CJS quality matter more than tool consolidation for a publishable library.

## Consequences

### Positive

- Clean separation between published library and demo surface.
- Workspace symlinks give playground HMR while editing the library.
- Turbo cache speeds up CI across `build`, `test`, `typecheck`, `lint`.
- Single `tsup.config.ts` handles ESM + CJS + types + CSS + subpaths.

### Negative

- Two package managers in the org (npm publish uses npm CLI under the hood) need consistent token handling.
- Engineers must learn Turborepo task graph semantics.

### Risks

- pnpm workspace symlinks confuse some bundlers in consumer apps. Mitigation: smoke test consumer integration via the playground using the published-style build (`pnpm pack` → `npm install ./bob-editor-*.tgz` in a sandbox app) before each minor release.
- `tsup` minor upgrades occasionally break CJS interop. Mitigation: pin `tsup` and run `publint` + `@arethetypeswrong/cli` on every PR.

## Implementation Notes

- `pnpm-workspace.yaml`: `packages: ['packages/*', 'apps/*']`.
- `turbo.json` declares `build`, `test`, `test:types`, `lint`, `typecheck`, `size`, with `dependsOn: ["^build"]` where relevant.
- `packages/bob-editor/tsup.config.ts` entries: `src/index.ts`, `src/plugins/index.ts`, `src/plugins/emoji.ts`, `src/plugins/mentions.ts`, `src/plugins/wordCount.ts`, `src/plugins/tableOfContents.ts`, plus a CSS-only entry.
- `package.json#files` whitelist: `["dist", "README.md", "CHANGELOG.md", "LICENSE"]`.
- CI: `ci.yml` (matrix Node 18/20/22 × React 18/19), `release.yml` (Changesets + provenance), `size.yml` (size-limit gate on PRs).
