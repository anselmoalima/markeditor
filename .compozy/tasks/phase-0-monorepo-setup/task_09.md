# Task 09: Changesets init (ignore apps/\*)

## Overview

Initialize Changesets so contributors declare version bumps inside PRs, and configure it to ignore every `apps/*` workspace so the playground is never released to npm. This is the versioning contract the release workflow (task_11) consumes.

<critical>
- ALWAYS READ the PRD and TechSpec before starting
- REFERENCE TECHSPEC for implementation details — do not duplicate here
- FOCUS ON "WHAT" — describe what needs to be accomplished, not how
- MINIMIZE CODE — show code only to illustrate current structure or problem areas
- TESTS REQUIRED — every task MUST include tests in deliverables
</critical>

<requirements>
- `@changesets/cli` MUST be installed at the root.
- `.changeset/config.json` MUST set `"access": "public"`, `"baseBranch": "main"`, `"changelog": "@changesets/cli/changelog"`, and `"ignore": ["playground"]` (and any other future `apps/*` workspace names).
- `.changeset/README.md` MUST be committed (auto-generated by `changeset init`).
- Root `package.json` MUST expose scripts `changeset`, `changeset:version`, `changeset:publish`.
- Creating a changeset against `markmd` MUST produce a markdown file under `.changeset/`.
- Running `pnpm changeset status` on a tree without changesets MUST exit non-zero (used by CI to remind contributors).
</requirements>

## Subtasks

- [x] 9.1 Install `@changesets/cli` at root and run `pnpm changeset init`.
- [x] 9.2 Edit `.changeset/config.json` to ignore `playground` and set `access: public`.

- [x] 9.4 Verify `pnpm changeset` creates a draft file and `pnpm changeset version` correctly bumps `packages/markmd` only.
