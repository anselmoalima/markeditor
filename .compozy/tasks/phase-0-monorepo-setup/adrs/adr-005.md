# ADR-005: Quality gates — ESLint flat config v9 + Prettier + Husky/lint-staged + size-limit + Codecov

## Status

Accepted

## Date

2026-05-22

## Context

The PRD demands zero-warning lint, strict TypeScript, bundle-size enforcement that breaks PRs, package-validity checks (`publint`, `attw`), and a coverage badge in the README. Local commits should fail fast before CI runs.

## Decision

- **Lint**: ESLint **flat config** (`eslint.config.js`) at repo root, ESLint v9+, plugins: `typescript-eslint`, `eslint-plugin-react`, `eslint-plugin-react-hooks`, `eslint-plugin-jsx-a11y`, `eslint-plugin-import-x`. Per-workspace overrides via flat config arrays.
- **Format**: Prettier 3 with shared `.prettierrc` at root.
- **TS**: `tsconfig.base.json` at root with `strict: true`, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`; each package/app extends it.
- **Git hooks**: Husky (`prepare` script auto-installs) + `lint-staged` runs `eslint --fix` + `prettier --write` on staged files.
- **Bundle size**: `size-limit` with `@size-limit/preset-small-lib`; PR comment via `andresz1/size-limit-action`. Limits enforced (see PRD §6.1).
- **Package validity**: `publint` + `@arethetypeswrong/cli` in CI and `prepublishOnly`.
- **Coverage**: Vitest v8 reporter → upload to Codecov via `codecov/codecov-action@v4` (token-less OIDC where possible); README badge.

## Alternatives Considered

### Alternative 1: Biome (replaces ESLint + Prettier)

- **Description**: Single Rust-based linter+formatter.
- **Pros**: Fast, one config.
- **Cons**: Plugin ecosystem (a11y, react-hooks rules) less mature than ESLint as of 2026.
- **Why rejected**: PRD explicitly cites ESLint + Prettier; a11y rules matter for AA target.

### Alternative 2: simple-git-hooks instead of Husky

- **Description**: Zero-dep hook installer.
- **Pros**: Smaller install.
- **Cons**: Smaller community, fewer recipes.
- **Why rejected**: Husky remains the path of least surprise.

### Alternative 3: No CI coverage upload

- **Description**: Skip Codecov; HTML artifact only.
- **Pros**: One less integration.
- **Cons**: No badge; harder to track regressions across PRs.
- **Why rejected**: Badge requested in PRD; Codecov free for OSS.

## Consequences

### Positive

- Single lint config; flat config is the future of ESLint.
- Pre-commit hooks catch issues before CI.
- Bundle bloat physically cannot land on `main`.

### Negative

- Husky adds a `.husky/` directory and a postinstall step; contributors using `--ignore-scripts` skip hooks (acceptable).

### Risks

- ESLint flat config plugin compatibility — some plugins still legacy. Mitigation: pin verified versions; have a fallback `eslintrc` only if a critical plugin lacks flat support.

## Implementation Notes

- `.husky/pre-commit` → `pnpm exec lint-staged`.
- `lint-staged` config in root `package.json`:
  ```json
  {
    "*.{ts,tsx,js,jsx}": ["eslint --fix", "prettier --write"],
    "*.{json,md,css}": ["prettier --write"]
  }
  ```
- `size-limit.json` lives in `packages/markmd/`; thresholds from PRD §6.1.
- Codecov action runs after `pnpm -r test` step in `ci.yml`.
