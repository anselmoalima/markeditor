# ADR-002: Build toolchain — tsup for dual ESM/CJS + d.ts + CSS

## Status

Accepted

## Date

2026-05-22

## Context

`packages/markmd` must publish a dual ESM/CJS bundle, emit `.d.ts` files matching both module systems, ship a bundled CSS importable as `markmd/styles`, preserve tree-shaking (`sideEffects: ["**/*.css"]`), and support subpath exports for plugins. Monaco, KaTeX, and Mermaid must remain code-split so bundle-initial stays <80KB gzip.

## Decision

Use **tsup** (esbuild-based) as the single bundler for the library:

- Entry points: `src/index.ts`, `src/plugins/*.ts`, `src/styles/index.css`.
- Outputs: ESM (`.mjs`) + CJS (`.cjs`) + types (`.d.ts` + `.d.cts`).
- `format: ['esm', 'cjs']`, `dts: true`, `splitting: true` (ESM only), `treeshake: true`, `external: ['react', 'react-dom', 'monaco-editor', 'katex', 'mermaid']`.
- CSS bundled via `tsup --loader .css=copy` plus an explicit entry; consumers import `'markmd/styles'`.
- `package.json` `exports` map covers `.`, `./styles`, `./plugins`, `./plugins/*`, `./package.json`.

## Alternatives Considered

### Alternative 1: Vite library mode

- **Description**: Use `vite build --lib` with `vite-plugin-dts`.
- **Pros**: Same tool as playground; good Rollup output.
- **Cons**: Dual ESM/CJS + dual `.d.ts`/`.d.cts` requires extra tooling (`rollup-plugin-dts`, separate config passes); slower than esbuild.
- **Why rejected**: tsup gives a one-line config for dual + types.

### Alternative 2: Rollup + plugins directly

- **Description**: Hand-rolled Rollup config.
- **Pros**: Maximum control.
- **Cons**: ~10× more config; maintenance burden.
- **Why rejected**: tsup is a thin Rollup/esbuild wrapper that solves 99% of cases.

### Alternative 3: tsc-only (no bundler)

- **Description**: Emit per-file ESM/CJS with `tsc`.
- **Pros**: Simplest; perfect tree-shaking.
- **Cons**: No CSS bundling, no `.cjs`/`.mjs` extensions in one pass, no inline plugin handling.
- **Why rejected**: CSS bundling requirement.

## Consequences

### Positive

- One config file produces ESM + CJS + types + CSS.
- esbuild speed keeps `pnpm build` under a few seconds.
- Subpath exports work via multiple entry points.

### Negative

- tsup is opinionated; advanced Rollup features require ejecting config overrides.

### Risks

- `.d.cts` generation must be validated by `@arethetypeswrong/cli` (attw) every release. Mitigation: include `attw` in `prepublishOnly` and CI.

## Implementation Notes

- Add `publint` and `attw` to CI and to `prepublishOnly`.
- `sideEffects: ["**/*.css"]` in `package.json`.
- Externalize React via `peerDependencies: { react: "^18 || ^19", react-dom: "^18 || ^19" }`.
- Lazy chunks (Monaco/KaTeX/Mermaid) use dynamic `import()` at the call site; tsup splits ESM automatically.
