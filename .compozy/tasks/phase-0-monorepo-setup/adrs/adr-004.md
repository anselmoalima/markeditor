# ADR-004: Versioning + publish — Changesets with NPM provenance via OIDC

## Status

Accepted

## Date

2026-05-22

## Context

The PRD requires SemVer-strict releases, automated changelog generation, supply-chain provenance, and a workflow where contributors declare version bumps inside PRs. The release pipeline must publish only `packages/markeditor`, never the playground.

## Decision

- **Changesets** (`@changesets/cli`) for version bumps and changelog generation.
- `.changeset/config.json` ignores `apps/*` so the playground never publishes.
- `release.yml` GitHub Actions workflow uses `changesets/action@v1` to open a "Version Packages" PR; merging that PR triggers `pnpm changeset publish` with `--provenance` via NPM OIDC.
- `prepublishOnly` runs `build + test + typecheck + lint + publint + attw + size`.

## Alternatives Considered

### Alternative 1: semantic-release

- **Description**: Commit-message-driven automated releases.
- **Pros**: Fully automated, no PR step.
- **Cons**: Requires strict commit discipline; harder to batch-release; less monorepo-friendly.
- **Why rejected**: Changesets gives explicit, reviewable version PRs and is monorepo-native.

### Alternative 2: Manual `npm version` + `npm publish`

- **Description**: Maintainer runs version+publish locally.
- **Pros**: Zero infra.
- **Cons**: No provenance, no changelog, error-prone.
- **Why rejected**: PRD §9 requires provenance + automation.

## Consequences

### Positive

- Provenance signature visible on npmjs.com (supply-chain trust).
- Changelog generated from PR-authored intents.
- Easy multi-package future (if `markeditor-react-native`, etc., emerge).

### Negative

- Contributors must remember `pnpm changeset` before merging feature PRs. Mitigation: changeset-bot or CI check that flags missing changesets.

### Risks

- OIDC misconfiguration could block release. Mitigation: dry-run release workflow on a test tag before v0.1.0.

## Implementation Notes

- `release.yml` permissions: `id-token: write`, `contents: write`, `packages: write`.
- `npm publish --provenance --access public` via Changesets.
- Tarball whitelist via `files: ["dist", "README.md", "CHANGELOG.md", "LICENSE"]`.
